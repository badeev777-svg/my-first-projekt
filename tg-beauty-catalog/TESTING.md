## 1. Что это за проект

**tg-beauty-catalog** — Telegram Mini App для записи к мастеру маникюра.  
Клиент видит каталог услуг, портфолио, отзывы и может забронировать время.

**Стек:** чистый HTML + CSS + JavaScript (ванильный, без фреймворков).  
**Нет бекенда.** Данные хранятся в `data.js` (статика) и `localStorage` браузера.  
**Нет тестов.** Всё тестирование — ручное.

---

## 2. Архитектура и ограничения

| Что | Как реализовано |
|-----|----------------|
| Данные об услугах, мастере, отзывах | Захардкожены в `tg-app/js/data.js` |
| Занятые слоты | Статический объект `busySlots` в `data.js` |
| Записи пользователя | `localStorage` ключ `tg_beauty_bookings` |
| Telegram SDK | `telegram-web-app.js` + браузерная заглушка в `app.js:13–85` |
| Бекенд / API | **Отсутствует** — `submitBooking` пишет только в localStorage |
| Бот-команды | **Отсутствуют** в Mini App |

> **Главное следствие:** нельзя «поломать» сервер — можно только проверить UI и логику фронтенда.

## 3. Настройка окружения

### Требования

- Любой современный браузер: Chrome 100+, Firefox 100+, Safari 15+, Edge 100+
- Для мобильного теста: iOS Safari или Chrome для Android
- Для теста в Telegram: аккаунт Telegram + права на создание бота через **@BotFather**

### Клонирование репозитория

```bash
git clone <URL репозитория>
cd tg-beauty-catalog
```

### Переменная BOT_TOKEN (нужна только для Telegram-бота)

```bash
cp .env.example .env
# Откройте .env и замените placeholder своим токеном от @BotFather:
# BOT_TOKEN=ваш_токен_здесь
```

> Для локального тестирования в браузере `.env` **не нужен** — Mini App работает без бота.

## 4. Запуск локально (браузер)

### Способ 1 — открыть файл напрямую

```bash
# Откройте в браузере:
tg-app/index.html
```

Просто перетащите `index.html` в окно браузера или используйте `File → Open`.

### Способ 2 — локальный HTTP-сервер (рекомендуется)

```bash
# Python 3
python -m http.server 8080 --directory tg-app

# Node.js (npx)
npx serve tg-app

# VS Code — расширение «Live Server», нажать «Go Live» на index.html
```

Откройте: `http://localhost:8080`

### Что работает в браузере

Файл `app.js` содержит заглушку Telegram SDK (строки 13–85). В браузере автоматически подключаются mock-объекты:

| Telegram API | Поведение в браузере |
|-------------|---------------------|
| `tg.ready()` / `tg.expand()` | Ничего не делают (no-op) |
| `tg.MainButton` | Синяя кнопка внизу экрана (`.tg-main-btn-fallback`) |
| `tg.BackButton` | Кнопка «← Назад» слева вверху (`.tg-back-btn-fallback`) |
| `tg.showConfirm(msg, cb)` | Стандартный браузерный `confirm()` |
| `tg.requestContact(cb)` | Вызывает `cb('+7 900 000-00-00')` с тестовым номером |
| `tg.HapticFeedback` | No-op (вибрация только на устройстве) |
| `tg.colorScheme` | `'light'` по умолчанию |
| `tg.CloudStorage` | `sessionStorage` браузера |

> Все кнопки, навигация и формы полностью работоспособны в браузере.

## 5. Запуск в Telegram Mini App

> Этот раздел нужен только для проверки реального Telegram-окружения.

### Шаги

1. Создайте бота: откройте [@BotFather](https://t.me/BotFather) → `/newbot`
2. Получите `BOT_TOKEN` и сохраните в `.env`
3. Задеплойте папку `tg-app/` на HTTPS-хост (Netlify, Vercel, GitHub Pages)
   - Netlify: папка публикации задана в `netlify.toml` → `publish = "tg-app"`
4. Привяжите Mini App: в @BotFather → `/newapp` → укажите URL вашего деплоя
5. Откройте бота в Telegram → нажмите кнопку запуска Mini App

### Что отличается от браузера

- Работает настоящий `tg.MainButton` (нижняя зелёная кнопка Telegram)
- Работает `tg.BackButton` (встроенная стрелка заголовка Telegram)
- Работает `tg.HapticFeedback` (вибрация на iPhone/Android)
- Работает `tg.requestContact()` — реальный диалог запроса контакта
- Тема (light/dark) берётся из настроек Telegram пользователя

## 6. Карта экранов и навигация

```
Нижнее меню (Tab Bar) — всегда видно на 4 основных экранах:
  [Каталог]  [О мастере]  [Портфолио]  [Мои записи]

Каталог ──────────────────► Детали услуги
                                │
                                ▼
                        Дата и время (Шаг 1)
                                │
                                ▼
                        Подтверждение (Шаг 2)
                                │
                                ▼
                         Успешная запись
                          │         │
                     "В каталог"  "В мои записи"
                          │         │
                       Каталог    Мои записи

О мастере ────────────────► Портфолио (фильтр по фото)
          └───────────────► Все отзывы
          └── MainButton ─► Каталог (переключает таб)

Портфолио ────────────────► Просмотрщик фото (fullscreen overlay)

Мои записи ──► "Перенести" ─► Дата и время (с заполненной услугой)
           └── "Записаться снова" ─► Дата и время
           └── "Записаться на приём" (MainButton) ─► Каталог
```

**Анимации переходов:**
- Таб → таб: `fadeIn` / `fadeOut` (200ms)
- Вперёд (push): `slideInFromRight` (250ms)
- Назад (pop): `slideInFromLeft` (250ms)

## 8. Сквозные сценарии (E2E)

### E2E-01: Полный путь записи

1. Открыть приложение → таб «Каталог»
2. Нажать «Покрытие гель-лак» (1 500 ₽, 90 мин)
3. Проверить детали. Нажать MainButton «Записаться»
4. Выбрать дату `2026-04-18` (есть свободные слоты)
5. Выбрать слот `10:30` (он свободен)
6. Нажать «Продолжить»
7. Нажать «Дать» — телефон подставился
8. В поле комментарий написать «Покрытие бежевое»
9. Нажать «Подтвердить запись»
10. Проверить: экран успеха, анимация, данные правильные
11. Нажать «В мои записи»
12. Проверить: запись появилась в «Предстоящие»

**Ожидаемый результат:** весь путь проходит без ошибок JS в консоли.

### E2E-02: Отмена записи

1. Выполнить E2E-01
2. Перейти в «Мои записи»
3. Нажать «Отменить»
4. В диалоге нажать «ОК» / «Да»
5. Проверить: запись исчезла
6. Открыть DevTools → `localStorage['tg_beauty_bookings']` — массив пустой или без этой записи

### E2E-03: Перенос записи

6. Убедиться: в «Мои записи» стала 2 запись (старая + новая)

## 9. Проверка тёмной темы

Тёмная тема включается автоматически если `tg.colorScheme === 'dark'`.

### Как включить в браузере

**Способ 1 — DevTools:**
1. Открыть DevTools (F12) → Console
2. Вставить и выполнить:
   ```javascript
   document.documentElement.classList.add('dark-theme');
   ```

**Способ 2 — системная тема:**
1. Переключить ОС в тёмный режим (Windows: Пуск → Параметры → Персонализация → Цвет → Тёмный)
2. Перезагрузить страницу — приложение подхватит `prefers-color-scheme: dark` через заглушку

### Что проверить

| # | Элемент | Ожидаемый вид (тёмная тема) |
|---|---------|----------------------------|
| TH-01 | Фон приложения | `#1C1C1E` (тёмно-серый) |
| TH-02 | Карточки услуг | `#2C2C2E` (чуть светлее фона) |
| TH-03 | Текст | `#F2F2F7` (почти белый) |
| TH-04 | Акцентный цвет кнопок | `#C9967A` (не изменяется, брендовый) |
| TH-05 | Разделители | `rgba(255,255,255,0.08)` (едва заметны) |
| TH-06 | Таб-бар | Тёмный фон, иконки светлые |
| TH-07 | Fullscreen просмотрщик | Чёрный оверлей |
| TH-08 | Обратное переключение | `classList.remove('dark-theme')` → всё возвращается |

## 11. localStorage — проверка персистентности

### Как проверить

1. Открыть DevTools → Application → Local Storage → `http://localhost:8080` (или `file://`)
2. Ключ: `tg_beauty_bookings`
3. Значение: JSON-массив объектов записи

## 12. Известные ограничения v1 (не баги)

> Следующие вещи намеренно не реализованы в текущей версии и **не являются багами**:

| Ограничение | Причина |
|-------------|---------|
| Записи не отправляются на сервер | Нет бекенда; данные только в localStorage |
| Занятые слоты не обновляются после записи | Статический `busySlots` в `data.js` |
| Телефон не валидируется | Не заявлено в v1 |
| Уведомления мастеру не отправляются | Нет бот-сервера |
| Оплата отсутствует | Вне скоупа v1 |
| Система лояльности отсутствует | Вне скоупа v1 |
| Мастер не может управлять слотами через UI | Управление через `data.js` напрямую |
| `initData` от Telegram не верифицируется | Нет бекенда для проверки подписи |

### Приоритеты

| Приоритет | Описание |
|-----------|----------|
| **P1 — Критический** | Приложение не запускается / краш JS / нет возможности завершить запись |
| **P2 — Высокий** | Навигация сломана / данные не сохраняются / неверные цены / недоступен целый экран |
| **P3 — Средний** | Визуальный баг мешает использованию / неверный текст / сломанный фильтр |
| **P4 — Низкий** | Косметический баг / опечатка / незначительное отклонение от дизайна |
