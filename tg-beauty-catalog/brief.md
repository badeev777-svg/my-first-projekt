# Brief: Telegram Mini App — Каталог услуг бьюти-мастера

> Версия: 1.0 (MVP)  
> Дата: 13.04.2026  
> Основан на: research.md

---

## Идея одной строкой

Личное приложение мастера внутри Telegram: клиент листает работы, видит услуги с ценами, выбирает время и записывается — не выходя из мессенджера.

---

## Принципы дизайна

| Принцип | Как выглядит |
|---|---|
| Нативный Telegram | Цвета из `themeParams`, `MainButton`, `BackButton` — всё родное |
| Минимум текста | Фото говорит за себя. Текст — только цена, название, время |
| Крупные кнопки | Высота CTA-кнопок ≥ 48px. Сетка слотов: ячейки ≥ 44×44px |
| Mobile-first | Всё проектируется для большого пальца, а не мышки |
| Русский язык | Интерфейс полностью на русском |
| Один экран — одна задача | Без перегруза. Каждый экран отвечает на один вопрос |

---

## Навигация (Bottom Tab Bar)

Постоянная нижняя панель на всех главных экранах (кроме шагов записи):

```
[ Главная ]  [ Услуги ]  [ Записи ]  [ О мастере ]
```

- Иконка + подпись под каждым табом
- Активный таб: акцентный цвет
- Высота таббара с учётом safe area (iPhone notch)

### Экран 1 — Главная (Home)

**Назначение:** первое впечатление, навигационный хаб.

**Элементы сверху вниз:**

```
┌─────────────────────────────────┐
│  [Аватар мастера]  Имя мастера  │  ← фото круглое, 48px
│  Специализация (nail / lash…)   │  ← серый текст, мелко
├─────────────────────────────────┤
│  ┌─────────────────────────┐    │
│  │  БАННЕР / АКЦИЯ         │    │  ← 16:6, скругление 16px
│  │  «Запись на апрель»     │    │     фото + текст поверх
│  └─────────────────────────┘    │
├─────────────────────────────────┤
│  Мои работы  →                  │  ← заголовок + «Все →»
│  [фото][фото][фото]             │  ← горизонтальный скролл
│   3 превью 1:1, скругление 12px │
├─────────────────────────────────┤
│  Услуги                         │  ← заголовок секции
│  [Маникюр]  [Педикюр]           │  ← chip-кнопки категорий
│  [Брови]    [Ресницы]           │     скругление 24px, 44px h
├─────────────────────────────────┤
│  Популярное                     │
│  [карточка услуги]              │  ← 2–3 карточки вертик.
│  [карточка услуги]              │
└─────────────────────────────────┘
```

**Действия:**
- Тап на баннер → переход на Акции (если нет акций — Каталог)
- Тап «Все →» рядом с «Мои работы» → Экран 9 (Портфолио)
- Тап на фото в превью → Экран 9 (Портфолио, открытое на этом фото)
- Тап на chip-категорию → Экран 2 (Каталог, отфильтрованный по категории)
- Тап на карточку услуги → Экран 3 (Детали услуги)

```
│  │ Покрытие гель-лак         │  │
```

### Экран 4 — Выбор даты и времени

**Назначение:** первый шаг записи — выбор слота.

**Элементы:**

```
┌─────────────────────────────────┐
│  [← BackButton]   Шаг 1 из 2   │  ← прогресс справа
├─────────────────────────────────┤
│  Покрытие гель-лак · 1 500 ₽   │  ← напоминание выбора
│  1ч 30м                         │
├─────────────────────────────────┤
│  Апрель 2026          [< >]     │  ← месяц + стрелки
│  Пн Вт Ср Чт Пт Сб Вс          │
│  -- -- 01 02 03 04 05           │
│  06 07 [08] 09 10 11 12         │  ← [08] = выбранный день
│  13 14  15  16 17 18 19         │  ← зачёркнутые = недоступны
│  …                              │
├─────────────────────────────────┤
│  Доступное время                │
│  [09:00] [10:30] [12:00]        │  ← сетка слотов 44×44px min
│  [13:30] [15:00]  ░░░░░░        │  ← серый = занято
│  [16:30] [18:00]                │
└─────────────────────────────────┘
              [  Продолжить  ]      ← MainButton (неактивна до выбора)
```

**Детали:**
- Доступные дни — обычный текст
- Недоступные (нет слотов) — серый, не кликабельный
- Выбранный день — кружок акцентного цвета
- Занятые слоты — серый фон, текст серый
- `HapticFeedback.selectionChanged()` при выборе слота
- `ClosingConfirmation` включён
- `disableVerticalSwipes()` включён

**Действия:**
- Тап на день → обновляет сетку слотов ниже
- Тап на слот → выделяет, активирует MainButton
- MainButton «Продолжить» → Экран 5 (Подтверждение)

**Назначение:** итоговый чек перед отправкой, второй (последний) шаг.

**Элементы:**

```

│  [← BackButton]   Шаг 2 из 2   │
├─────────────────────────────────┤
│  Ваша запись                    │  ← Bold 20px
│  ┌───────────────────────────┐  │
│  │ 📋  Покрытие гель-лак     │  │
│  │ 📅  Вт, 15 апреля         │  │
│  │ 🕐  13:30  ·  1ч 30м      │  │
│  │ 💰  1 500 ₽               │  │
│  └───────────────────────────┘  │
├─────────────────────────────────┤
│  Ваш телефон для связи          │
│  [+7 (___) ___-__-__]  [Дать]  │  ← кнопка requestContact()
├─────────────────────────────────┤
│  Комментарий (необязательно)    │
│  ┌───────────────────────────┐  │
│  │ Напишите пожелания…       │  │  ← textarea, max 200 символов
```

**Действия:**

**Элементы:**

```
│                                 │
│       [Lottie анимация]         │  ← checkmark или confetti
│         ✓ Вы записаны!         │  ← Bold 24px, центр
│                                 │
│  ┌───────────────────────────┐  │

│  │ Вт, 15 апреля · 13:30     │  │
│  │ 1 500 ₽                   │  │
│  └───────────────────────────┘  │
│                                 │
│  [  Поделиться с подругой  ]    │  ← outlined кнопка 48px
│  [  В мои записи           ]    │  ← outlined кнопка 48px
│                                 │
```

**Действия:**

## Все экраны и переходы (сводная таблица)

| # | Экран | Откуда попасть | Куда ведут действия |
|---|---|---|---|
| 1 | Главная | Старт / таб | Портфолио, Каталог (по категории), Детали услуги |
| 2 | Каталог | Таб, Главная | Детали услуги |

## Чего НЕ будет в версии 1.0

| Что | Почему отложено |
|---|---|
| Оплата депозита | Требует ИП / самозанятость + подключение платёжного провайдера — блокер для MVP |
| Несколько мастеров | Сначала отрабатываем модель для одного — проще тестировать |
| Программа лояльности | Баллы нужны когда клиент уже возвращается — работа для v2 |
| Оставить отзыв | Отзывы в v1 отображаем статично (захардкожены или через бот) |
| Слайдер «до/после» | Полезно, но не критично для MVP — в v2 |
| Акции / промокоды | Маркетинговый инструмент для уже работающего продукта |
| Поиск по услугам | Достаточно фильтров по категориям пока услуг < 20 |
| Push-напоминания | В v1 напоминание через бот-сообщение вручную; автоматика — в v2 |
| Панель мастера (admin) | Мастер управляет слотами через бот-команды, не через Mini App |
| Full-screen режим | Bot API 8.0+ — подключим в v2 после отработки основного флоу |
| Мультиязычность | Только русский в v1 |

## Технические требования

### Стек (рекомендуемый)
- **Frontend:** React + TypeScript + Vite
- **Telegram SDK:** `@telegram-apps/sdk` или `@twa-dev/sdk`
- **Стили:** CSS Modules или Tailwind (без UI-библиотек — нативный вид через `themeParams`)
- **Изображения:** WebP, lazy loading, CDN
- **Анимации:** Lottie (только для экрана успеха и пустых состояний)

### Telegram API — обязательно использовать
```
WebApp.MainButton          — CTA на шагах записи
WebApp.BackButton          — навигация назад
WebApp.HapticFeedback      — отклик на выбор слота и успех
WebApp.CloudStorage        — черновик записи (persist между сессиями)
WebApp.showConfirm()       — диалог отмены записи
WebApp.requestContact()    — получить телефон клиента
WebApp.enableClosingConfirmation()  — на шагах 4–5
WebApp.disableVerticalSwipes()      — на шагах 4–5
WebApp.themeParams         — цвета из Telegram темы
```

### Требования к производительности
- FCP (First Contentful Paint) ≤ 1.5 сек
- Skeleton-экраны при любой загрузке > 200мс
- Изображения в сетке: thumbnail max 300px, WebP
- Полноэкранное фото подгружается только при открытии

### Безопасность
- Верификация `initData` на сервере (HMAC-SHA256)
- Все запросы к API только через HTTPS
- Телефон клиента не хранится в `CloudStorage`
